Functional Description
The SMStoMMS application forwards an incoming SMS message as a MMS message.
If the the SMStoMMS feature feature is enabled for the destination number (SUBSCRIBER_B), then every incoming SMS message of a friend (SUBSCRIBER_A) to SUBSCRIBER_B’s phone number will be forwarded as a MMS message.
The MMS message shall contain the original SMS text and media (an image).

For example:
Jack: 0612345678
Jill: 0698765432
Jack (a friend of Jill) sends a SMS message to Jill, with content: I’ll be late for dinner. Jill will receive a MMS message on her phone with text: I’ll be late for dinner again and an image

A Subscriber
An A_Subscribers is a person who initiates a certain action (here sending a SMS message to another person). The applications can have ceveral A subscribers.

B Subscriber
The B_Subscriber mentioned is the person who is registered to an application. This means that the phone number is set in the application to be the destination address, on which the application triggers (e.g. when a message is received for this number).


Configuration
The following settings have to be specified in the configuration file:

Variable Description
subscriber.phone Phone number of the subscriber for whom the service will be activated
friend.0 - friend.3 Phone numbers of which the receiving SMS will be forwarded as MMS (note that for each friend a jpg image must exist in the directroy content with the same phonenumber, e.g. 123.jpg if a phone number is 123)
originatingTrigger The trigger value of the originating addresses, the friends (e.g. ??? if all friends numbers consitst out of 3 digits)
serviceCode A operator specific value (of 2 digits), on this value there will also be triggered
billing Billing string (agreed with the operator) to specify how billing of the MMS is handled
responseRequired Specifies if a response is requested on the MMS (true / false)
deliveryNotification Specifies if a delivery notification of the MMS is desireable (true / false)
validityTime Time the MMS message stays valid (e.g. 2010-01-01 00:00:00.000)

See the Common Usage - General Configuration also.
